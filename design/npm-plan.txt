repo state. Supersedes the original conversational plan (full history
in git). The original was drafted before the following work landed:

- Signal-type registry (593574c) — signal.Make now panics on unregistered types

## Locked decisions

- **Q1 Manifests.** `package.json` + `package-lock.json` only.
  yarn.lock / pnpm-lock.yaml deferred to post-v0.1.
- **Q2 Validator.** DONE. ValidateCanonicalURI admits scoped-package
  URIs since #78's printable-ASCII refactor. Pinned with test cases
  in `TestValidateCanonicalURI_Accepts` for `@types/node`,
  `@nestjs/core`, `@angular/core`.
- **Q3 Verification.** Trust the registry's `repository.url` claim
  without pre-verifying existence (one extra API call per package is
  expensive and the downstream github collector handles absence
  gracefully). Ship OIDC trusted-publishing attestation presence as
  the provenance signal — it's cryptographically verifiable and
  directly addresses the axios attack pattern. Defer git-tag-
  correspondence and tarball-diff to post-v0.1; they require cross-
  collector orchestration worth its own design pass.
- **Q4 Decomposition.** Three commits (Phase A / B / C), not one
  megacommit. Each is independently reviewable and leaves the test
  suite green.
- **Q5 Signals.** Flat metadata, 5 or 6 entries (see open question
  below): `last_publish`, `maintainer_count`, `postinstall_present`,
  `trusted_publishing`, `weekly_downloads`, optionally
  `dependent_count`.
- **Decision (a) Repo resolution.** Lives in analyze.go orchestration,
  not inside the npm collector. Provider answers "what's the repo for
  this package?"; orchestrator stamps it on the entity; collectors
  work against the resolved entity.
- **Decision (b) Default-on.** Collector is default-enabled. Scheme-
  filtered (acts only on `pkg:npm/*` entities) so no false triggers.
- **Decision (c) Traffic.** 3 external calls per npm analyze is
  acceptable for v0.1. Prefer bundling where feasible.

## Architecture shape

Three orthogonal pieces:

1. **Manifest parsing** — `internal/manifest/npm/`. Returns
   `[]manifest.Dep` identical to gomod's shape so survey can consume
   it without special-case code.
2. **Registry client** — `internal/signal/registry/npm/client.go`.
   HTTP to `registry.npmjs.org` (main metadata) and `api.npmjs.org`
   (downloads). Models `signal/github/client.go` for defense-in-depth.
3. **Signal collector** — `internal/signal/registry/npm/collector.go`.
   Implements `signal.Collector`; scheme-filtered to `pkg:npm/*`.

Orchestration in `analyze.go`:

- `profile.ResolveTarget(cmd.Target)` → canonical URI + scheme
- If `scheme == pkg` + `ecosystem == npm`: call npm client once for
  `repository.url`, structurally validate, stamp on entity
- `collectorsFor(entity)` returns the applicable subset based on
  entity type + resolved URL

## Phase A — architecture shift + first end-to-end signal (~500 LOC)

Goal: prove analyze → npm-collector → store plumbing end-to-end with
a single signal. This phase pays down the analyze.go tech debt
(ResolveTarget adoption + collector-list generalization) as a
byproduct, independent of the npm-signals-quality conversation.

### A.1 analyze.go: adopt ResolveTarget

- Replace the direct `NormalizeGitHubRepoInput` call at
  `analyze.go:73` with `profile.ResolveTarget(cmd.Target)`.
- Branch entity creation on `resolved.Scheme`:
  - `repo:*` → `EntityProject` + `URL=https://<platform>/<owner>/<name>`
    (preserves existing behavior exactly)
  - `pkg:npm/*` → `EntityPackage` + `Ecosystem="npm"` + `URL=""`
    (URL populated in A.5 after repo resolution)
- Tests: every existing github analyze case stays green; one new
  test for a `pkg:npm/` target that exercises the new branch.

### A.2 collectorsFor: gate by entity type

Current: `[github, git-local-clone]` for all entities; git mandates
`--path`/`--clone` unconditionally.

New dispatch:

- `EntityProject` + github URL → `[github, git-local-clone]` (git
  requires `--path`/`--clone` as today)
- `EntityPackage` + `Ecosystem=npm` → `[npm-registry]`; git-local-
  clone skipped; `--path`/`--clone` NOT required
- `EntityPackage` that resolves to a github repo (A.5) →
  `[npm-registry, github, git-local-clone]`; git-local-clone still
  requires `--path`/`--clone` to run but is now optional

The sentinel `ErrCloneRequired` must only fire when a git-hosted
entity is analyzed without `--path`/`--clone`. npm-only analysis
doesn't trigger it. Update tests for the 5 existing sentinel errors
accordingly.

### A.3 npm registry client

`internal/signal/registry/npm/client.go`. Single method for Phase A:

```
GetPackage(ctx context.Context, name string) (*registryPackage, error)
```

Defense-in-depth invariants (all tested):

- HTTPS-only redirect policy — refuse http downgrade (#89 pattern)
- Max response size 10MB (npm metadata can be multi-MB legitimately)
- Error string NEVER contains response body (#93 pattern)
- `json.Decoder.DisallowUnknownFields()` on `registryPackage` struct
  — strict decode surfaces registry-schema drift the same way
  yaml-strict-mode surfaces analyst-output drift (220ed34 lesson)
- Context propagation + 60s per-request timeout
- Package-name validation before URL construction — allowlist
  `[A-Za-z0-9._-]` plus a single leading `@<scope>/` for scoped
  names; reject anything else before any HTTP activity

`registryPackage` models only what Phases A+B read: `time` (last
publish), `maintainers`, `versions[<dist-tags.latest>].scripts.postinstall`,
`dist.attestations` (OIDC), `repository.url`. Unused fields are not
modeled (DisallowUnknownFields → any drift is a test failure we'd
want to know about).

Client tests use httptest server returning hand-crafted minimal JSON:
happy path, size-cap, HTTPS downgrade rejection, error-body
sanitization, malformed-name rejection pre-HTTP, context cancellation.

### A.4 npm collector + first signal

`internal/signal/registry/npm/collector.go` implements
`signal.Collector`:

- `Name() → "npm-registry"`
- `Collect(ctx, entity)` — no-op (empty result) for any entity whose
  `CanonicalURI` doesn't start with `pkg:npm/`; otherwise fetch via
  client and emit signals

Phase A ships ONE signal: `last_publish` (Group=vitality,
ForgeryResistance=medium). Must land with:

- Entry in `internal/signal/types.go`'s `signalTypeRegistry` with
  Group, ForgeryResistance, Description, Caveats populated
- Unit tests: happy-path extraction, missing `time` field →
  absence signal, non-npm entity → empty result

### A.5 Repo resolution in analyze.go

After entity creation, if `EntityPackage` + `Ecosystem=npm`:

1. Call `npm.Client.GetPackage(ctx, name)`
2. Extract `repository.url` (handle both string and `{type,url}`
   object shapes — npm allows both)
3. Structurally validate: must parse to a URL with host matching
   an allowlist (`github.com` for v0.1 — other platforms in later
   work)
4. Stamp on entity as `URL`

If repository is absent, malformed, or non-allowlisted: leave
`URL=""`. github collector is either skipped by A.2's dispatch or
records absences when run — both acceptable.

Don't pre-verify the github repo exists (Q3 locked). Failures
surface downstream as collector absences.

### A.6 End-to-end functional test

`cmd/signatory/functional_test.go` —
`TestFunctional_AnalyzeNpm_EndToEnd`:

- Target: `pkg:npm/express`

## Phase C — manifest parsing + survey routing (~300-400 LOC)

Goal: `signatory survey` works against npm projects. Closes the
dogfood gap the /analyze skill already claims.

### C.1 internal/manifest/npm/ parser

- `Parse(path) (manifest.ProjectInfo, []manifest.Dep, error)`
- Input: `package.json` (required) + `package-lock.json` v3 (if
  present — used to resolve transitive versions)
- Four dep classes: `dependencies`, `devDependencies`,
  `peerDependencies`, `optionalDependencies`. All become
  `manifest.Dep` entries with `Direct=true`; class captured via
  the `Ecosystem` subfield or a new tag if justified
- Transitive deps (from lockfile): `Direct=false`
- CanonicalURI:
  - Unscoped: `pkg:npm/<name>`
  - Scoped: `pkg:npm/@<scope>/<name>`
- Non-registry deps (file:, git:, npm:alias): `Ecosystem=
  "npm-local"`, empty CanonicalURI (mirrors gomod's
  `go-local-replace` pattern)
- `package-lock.json` v3 only for v0.1 (npm 7+). v1/v2 and workspace
  entries: gracefully parsed-past or skipped; not an error

testdata/ fixtures (check in as real package.json + package-lock.json
pairs for determinism):

- `simple/` — 2 direct deps + ~5 transitive
- `scoped/` — includes a `@types/node` direct dep
- `mixed/` — all four dep classes populated
- `no-lockfile/` — package.json only

Parser edge cases to cover:

- Empty dependencies map
- `repository` as string vs object
- Missing `repository` field
- Workspace-style lockfile entries (ignore gracefully)

### C.2 manifest.Detect routing

- Append `package.json` to the candidate list in
  `internal/manifest/detect.go`
- Tie-breaking order (if both go.mod and package.json exist):
  first-found wins; document the expectation. v0.1 doesn't need
  explicit tie-breaking because real projects are single-ecosystem.

### C.3 survey.Run routing

- `internal/survey/survey.go` `parseManifest` switch: add
  `"package.json"` case routing to `npmmanifest.Parse`
- Tests: survey against each testdata fixture; summary renders
  correctly; tiers resolve against store

### C.4 Dogfood

- Live run: `signatory analyze pkg:npm/vitest --refresh`
- Optional: add a small npm testdata project; run
  `signatory survey` against it
- /analyze skill routing check: does `/analyze pkg:npm/vitest`
  route through the skill correctly?

## Security invariants (apply to every phase)

- `ValidateCanonicalURI` at persistence boundary — unchanged
- HTTPS-only redirect in npm client (#89)
- Max response size cap (10MB per github-client pattern)
- Error sanitization: response body NEVER in error string (#93)
- Package-name validation before URL construction (allowlist +
  scoped-name grammar)
- Context cancellation propagation tested (#87 pattern)
- `json.DisallowUnknownFields()` for every registry-response struct
- `signalTypeRegistry` entry for every new signal type (otherwise
  `signal.Make` panics at first emission)

## Test discipline

- Table-driven unit tests with explicit `name` fields and
  `t.Run(tc.name, …)` per case
- TDD for security-adjacent fixes: failing test before the fix
- Writer injection (per survey's pattern) for any stdout output;
  no global `os.Stdout` redirection — breaks `t.Parallel()`
- Functional tests via `httptest`, never live npm/github

## Out of scope (post-v0.1)

- yarn.lock / pnpm-lock.yaml
- Git tag correspondence provenance signal
- Tarball-vs-source content diff
- Authenticated npm registry requests (tokens)
- Private/scoped registries
- Cross-ecosystem correlation
- TTL cache on registry responses (ROADMAP should-do)
